# LLM Wiki를 업그레이드한 Graphify 개념 + 실습 가이드

## 요약
이 영상은 LLM Wiki의 업그레이드 버전인 Graphify를 소개하고 실제 사용 방법을 설명합니다. Graphify는 AST(Abstract Syntax Tree) 이론을 적용하여 지식을 기계적으로 전처리함으로써 LLM Wiki 대비 약 5배 이상의 토큰 절감을 달성합니다. 하이브리드 전략으로 뼈대만 추출하여 에이전트에 전달하면 더 효율적인 구조화와 처리가 가능해집니다. 실습을 통해 가상환경 설정, Graphify 실행, 그래프 HTML 생성, 지식베이스 쿼리까지의 전체 프로세스를 단계별로 보여줍니다.

## 인사이트
- Graphify는 단순히 LLM Wiki의 확장이 아닌 전혀 다른 아키텍처로, 토큰 낭비를 최소화하기 위해 전처리 단계에서 기계적 분석을 수행하여 LLM에 최적화된 형태로 정보를 제공합니다.
- AST 기반의 전처리로 인해 병렬 작업이 가능해지므로 처리 속도가 빠르고, GPT-4o Mini 같은 저가형 모델을 활용할 수 있어 API 비용 절감 효과가 크다는 것이 Graphify의 핵심 경쟁력입니다.
- Graphify를 통한 지식베이스 구축 후 특정 명령어(graphify-codex install 등)로 에이전트를 설정하면, 해당 지식 그래프 범위 내에서만 답변하도록 제약할 수 있어 할루시네이션 방지 및 정확도 향상이 가능합니다.
- 그래프 HTML 시각화는 추가 기능이지만, 노드와 연결 관계를 한눈에 파악하여 지식 구조가 제대로 구축되었는지 검증하고 누락된 연결고리를 발견할 수 있게 도와줍니다.
