# GStack + Superpowers로 클로드코드 완전히 달라진 사람의 후기

## 요약
메이커 에반이 Claude Code의 플러그인 중 GStack과 Superpowers 두 가지만 남기고 사용하면서 얻은 개선 경험을 공유한다. 오피스 아워스로 아이디어를 정리하고, 디자인 리뷰로 결과물을 검증하며, 서브 에이전트와 워크스페이스 분리 기능으로 효율적으로 작업하는 구체적인 워크플로우를 소개한다. 많은 기능보다 자연스러운 흐름과 사용자 맞춤형 컨트롤이 가능한 플러그인 조합이 더 효과적임을 강조한다.

## 인사이트
- Claude Code 사용 시 즉시 개발하기보다는 Office Hours를 통해 아이디어를 먼저 구체화하는 초반 단계가 매우 중요하며, 이를 통해 후반부 수정 작업을 크게 줄일 수 있다.
- 복수의 플러그인 기능을 한 번에 사용하는 것보다 GStack과 Superpowers처럼 워크플로우와 자연스럽게 연결되는 최소한의 도구 조합이 더 높은 품질의 결과물을 만든다.
- Brainstorming → Writing Plans → Sub-agent Development → Workspace Separation이라는 명확한 순서의 워크플로우를 따르면 프로젝트 중간에 방향이 흔들리지 않고 일관성 있는 결과를 얻을 수 있다.
- 자동 기록(auto-logging)과 워크스페이스 분리 기능으로 다중 작업 처리 시 각 프로젝트의 독립성을 보장하고 작업 중단 없이 연속성 있게 진행할 수 있다.
- Design Review와 Mockup 생성을 통해 피그마 같은 별도 디자인 도구 없이도 실시간으로 UI/UX를 수정하고 디렉팅할 수 있어 개발 속도가 대폭 향상된다.
