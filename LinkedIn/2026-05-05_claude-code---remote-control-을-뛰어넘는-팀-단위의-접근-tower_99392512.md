Claude Code - Remote Control 을 뛰어넘는 팀 단위의 접근. Tower

100여 명을 대상으로 한 Claude Code 강연과 5,000명 이상의 청중과 함께한 AI 강연을 진행하며 한 가지 확신이 생겼습니다.

모든 기업은 견적서, 계약서, 회의록, 소스 코드, 디자인 시안, 고객 리스트와 같은 방대한 데이터를 보유하고 있습니다. 하지만 이 귀중한 자산들은 대부분 폴더 속에 파편화된 채 방치되어 있습니다. 파일은 존재하지만, 그 안에 담긴 '맥락'은 사람의 머릿속에만 희미하게 남아 있을 뿐입니다. "왜 이런 결정을 내렸는지", "이 수치는 어디서 유도되었는지", "디자인 수정 과정에서 어떤 논의가 오갔는지"는 파일만 봐서는 결코 알 수 없습니다.

파일과 데이터가 기업의 뼈대라면, 그 뼈대에 의미와 맥락이라는 살을 붙이는 것이 진짜 '일'의 핵심입니다. 그리고 제가 경험한 바로는, 이 작업을 완벽하게 수행할 수 있는 AI는 단 하나뿐이었습니다.

바로 터미널 환경에서 모든 파일을 읽고, 구조를 파악하며, 맥락을 깊이 있게 해석하는 Claude Code입니다.

수많은 AI 도구를 테스트해 보았지만, 파일 시스템 위에서 자유자재로 움직이며 기업 고유의 맥락을 이해하는 도구는 이것이 유일했습니다. 저는 여기서 미래를 보았습니다.

하지만 문제는 그 강력한 미래가 '터미널'이라는 좁은 창에 갇혀 있다는 점이었습니다.

동료 기획자, 디자이너, 클라이언트, 그리고 AI의 도움을 필요로 하는 주변 사람들에게 CLI(명령어 라인 인터페이스)는 여전히 거대한 장벽입니다.

또한, 내 컴퓨터를 닫는 순간 모든 것이 멈춥니다. 공들여 설정한 스킬, 워크스페이스의 맥락, 그동안 쌓아온 데이터는 오직 한 대의 기기에만 종속됩니다. 세션을 팀원과 공유할 수도, 누군가 내 작업을 이어받을 수도 없습니다. 팀이 함께 맥락을 쌓아가며 AI를 더 똑똑하게 길러내는 협업은 불가능했습니다.

제가 꿈꾼 경험은 이런 것이었습니다.

"예전 견적서 디자인을 참고해서, A 폴더에 있는 신규 고객용 견적서랑 PPT를 전부 제작한 뒤 내 메일로 보내줘."

터미널을 모르는 팀원 누구라도, 브라우저를 통해 이 한마디로 업무를 처리할 수 있는 환경 말입니다.

그래서 지난주 토요일, 'Tower'를 만들기 시작했습니다.

Tower는 하나의 공유 파일 시스템 위에 Claude Code의 강력한 성능을 결합하여, 팀 전체가 브라우저에서 손쉽게 접근할 수 있도록 설계된 시스템입니다.

20여 종의 AI 스킬 내장: 브레인스토밍부터 TDD, 디버깅, 코드 리뷰, 기획까지 즉시 활용 가능
3계층 메모리 시스템: 팀의 결정 사항, 맥락, 학습 데이터를 AI가 지속적으로 기억
역할 기반 권한 관리: 관리자에게는 전권을, 일반 사용자에게는 안전한 가드레일을 제공
어디서나 접근 가능한 환경: 한 번의 배포로 장소와 기기에 상관없이 접속
이것은 단순한 데모나 껍데기(Wrapper)가 아닙니다. 팀 전체가 하나의 파일 시스템 위에서, 하나의 AI와 함께 맥락을 공유하며 일하는 '살아있는 워크스페이스'입니다.

직접 사용해 보니 확신이 생겼습니다.

과거에는 AI가 그저 외부 도구에 불과했다면, 이제는 다릅니다. 스스로 진화하고 지식을 축적하며 업무 흐름을 정리하는, 팀원 모두가 언제든 도움을 받을 수 있는 든든한 동료가 생겼습니다.

파일 시스템이 기업 그 자체가 될 수 있겠단 비전이 강해졌습니다. 그 시스템에 생명력을 불어넣는 AI가 있다면, 더더욱 그렇습니다. 저는 최근에 굳이 RAG, ontology 가 필요한가 하는 생각을 했습니다. 온톨로지보다, 잘 만든 research skill 하나가 파일들을 훑어보며 직접 정리한 메뉴얼이 훨씬 강력한 경우가 많기 때문입니다. 세상에. 온톨로지도 옛날 말이라니. RAG도 구시대적 기술이라니.

경험해보지 않은 분들은 모를 겁니다. bash 로 시스템 전체를 죽을 때까지 뒤져내는 claude code 의 권능을 말입니다.

전체 시스템을 오픈소스로 공개합니다. 아직 부족한 점이 많지만, 새로운 협업의 시대를 여는 좋은 시작점이 되기를 바랍니다.
