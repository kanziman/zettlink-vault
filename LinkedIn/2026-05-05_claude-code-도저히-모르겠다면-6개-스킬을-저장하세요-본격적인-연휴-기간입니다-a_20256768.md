Claude Code 도저히 모르겠다면, 6개 스킬을 저장하세요.

본격적인 연휴 기간입니다.
AI 가 여전히 낯선 분들은 이 3가지만 하세요.
→ ChatGPT Plus 구독하기 (Claude Pro 도 좋지만 많이 사용하기 어려움)
→ Codex App 설치하기
→ Skill 작성 방법 확인 후 하나 만들어보기

마지막 내용에 대해 다루어 보겠습니다.
Y Combinator CEO Garry Tan이 "Thin Harness, Fat Skills" 원칙을 공개했습니다. 딥하게 들어가면 한도 끝도 없지만 대부분 에이전트의 실제 생산성 차이는 모델이 아니라 스킬에서 갈립니다. 

제가 직접 다 써보고 6개로 추렸습니다.
중복해서 사용하기 보다는 하나를 선택해서 사용하세요.

❶ Anthropic 공식 스킬은 스킬 작성의 표준입니다 

https://lnkd.in/gan2vtXH

스킬의 스펙 가이드라인과 템플릿이 포함되어 있습니다. 
SKILL.md를 어떤 형식으로 써야 할지 모르겠다면 여기서 구조를 따라가세요.

❷ Karpathy 스킬은 CLAUDE.md 하나로 코드 품질을 잡습니다 

https://lnkd.in/gbpaWFua

Andrej Karpathy 가 관찰한 LLM 코딩 실패 패턴을 파일 한 장에 담았습니다. 폴더 구조 없이 루트에 넣으면 끝이에요. 스킬의 가장 가벼운 형태입니다.

❸ Superpowers는 개발 방법론 전체를 스킬로 엮었습니다 

https://lnkd.in/gwas6su7

브레인스토밍 → 스펙 → 구현 → 서브에이전트 실행 → 2단계 코드 리뷰까지 전체 사이클을 스킬로 구성했습니다. 일명 태스크 주도 개발(TDD)을 red-green-refactor 로 강제하여 AI 가 의도하지 않은 방향으로 가지 않도록 만듭니다.

❹ GStack은 에이전트에게 역할을 부여합니다 

https://lnkd.in/gR4E47Vc

Garry Tan 본인의 세팅을 공개한 레포입니다. 
CEO, 디자이너, QA까지 23개 도구가 역할별로 나뉘어 있어요. 
"Thin Harness, Fat Skills"를 만든 사람의 실전 구현체입니다. 
스킬 하나 하나보다 전체를 어떻게 조합하는지 보기 좋습니다.

❺ GSD는 context rot을 스킬로 해결합니다 

https://lnkd.in/giDxZHB6

discuss → plan → execute 루프를 돌면서 각 단계를 새 컨텍스트에서 실행합니다. 현재 상태를 별도의 파일로 저장해서 장기 프로젝트에서 결과 품질이 떨어지는 문제를 잡아요. 

하지만 매 대화마다 약 12K 토큰이 고정적으로 사용됩니다. 따라서 context window가 작은 로컬 모델에서는 주의하세요.

❻ Matt Pocock 스킬은 코딩 전에 정렬부터 시킵니다 

https://lnkd.in/giVJsF9i

grill-me가 가장 유명합니다. 코드 작성 전에 에이전트가 요구 사항을 질문하며 모든 수정 포인트를 확인해요. caveman은 토큰 사용량을 약 75% 줄이고 domain-model 은 도메인 용어를 정리해서 에이전트와 같은 언어로 대화할 수 있도록 합니다. 스킬 엔지니어링이 처음이라면 이 레포의 README 를 읽어보시는 것을 강력하게 추천합니다.

Anthropic은 스킬 작성 표준. 
Karpathy는 강력한 1-file 스킬의 예시. 
Superpowers는 방법론을 도입한 스킬. 
GStack은 역할론을 도입한 스킬. 
GSD는 장기 프로젝트 관리를 위한 스킬. 
Matt Pocock은 용어 정렬과 피드백 루프가 확립된 스킬. 

성격이 전부 다릅니다.
스킬 관련 콘텐츠 앞으로도 계속 정리해보겠습니다.
