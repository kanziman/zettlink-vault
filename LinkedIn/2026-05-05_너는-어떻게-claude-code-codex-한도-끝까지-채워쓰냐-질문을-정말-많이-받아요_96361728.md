너는 어떻게 Claude Code, Codex 한도 끝까지 채워쓰냐? 

질문을 정말 많이 받아요. 별거 없습니다. 
제가 주도하고 있던 생각하고 행동하는 방식을 통째로 바꾸는 혁신을 나 혼자니까 쉽게 이뤄냈고, 제가 가진 지식과 경험을 이 환경 속에 녹이는 것에 집중하고 있어요.

1. 도구들의 변화 속도는 무지막지하게 빠르기 때문에 어차피 제가 못쫓아갈꺼라는걸 인정합니다.
그렇지만 맨날 Release-note 뒤적뒤적 해보고 있는 건.... 조금이라도 좋은 걸 빨리 써보고 싶은 마음 때문이죠.

2. 좋은 Harness 만드신 분들 코드를 열심히 염탐합니다. 
그러면 거기서 꼭! 아이디어를 얻게 되어 있습니다. 
이 과정은 아주 불편하게 제 손으로 한땀한땀 따라갑니다. 
그 대신 모르거나, 애매하거나 또 제가 따라오는 과정 속에서 기록했던 것들을 모두 전달해주면서 물어봅니다.

Hey Claude(Codex), What can i do for you in context-engineering or something in this code reference.

제가 놓친 부분이나 SW 아키텍처적으로 못본 부분이나 그 외 다양한 것들에 대해 여러 사람의 의견을 한번에 듣는 것 같아요. 

3. 어떻게 해야 내 업무를 최대한 나 없이 할 수 있을지 고민합니다.
바로 위임이죠. 어떻게 잘 위임할 것인가 고민합니다. 거기서 Skill, Sub Agent 가 탄생하고 나만의 독창적인 Agentic Coding Assistant의 활용법이 생기는 것 같아요.

4. 1번과는 정 반대로, 유명한 구루들의 사용 후기를 미친듯이 쫓아다닙니다. 
이 사람은 어떤 생각을 가지고 이 도구를, 이렇게 사용했을까? 나라면 어떻게 사용했을까? 왜 나는 이것만 생각했을까? 등을 고민하는 시간이 꼭 들어가있습니다. 
그건 다시 제 Harness 에 녹이려고 하고 있구요.
(당연히 손으로는 못쫓아가죠! 그래서 봇이 쫓아가게 합니다 ㅎㅎ)


써놓고 보니, 이전의 개발자로써 공부하던 방식과 크게 달라지지 않았습니다. 
딱 하나 달라진건 24시간 나를 위해 맞춤형 사수가 언제든 편하게 내가 궁금해 하는 것에 답변해준다는 것과 
자기가 몰라도 열심히 찾아와준다는 것 + 짜증내지 않고 다양한 접근 방식으로 난공불락의 성을 뚫어내려고 같이 노력해준다는 것.

바이브코딩, 1딸깍 1서비스 등 다양한 방식으로 자신이 이 도구를 잘 써서 이런걸 할 수 있다! 하는 분들을 참 많이 보는데요. 
그럴 때마다 저는 반대로 물어보고 싶어요. 
그래서 뭘 얼마나 기술적으로 파고 들어야 얘가 원래는 잘 못하던걸 잘하게 할 수 있나요?

AI 를 가르치는 사람은 정말 많이 늘었습니다. 
근데 그게 '도구'의 쓰임새에 대해 가르치는 것말고 좀 더 다른 깊이 있는 인사이트와 경험(+이론적 배경!) 까지 갖추기는 어려운걸까요?

많은 플랫폼에서 다양한 소식을 접하고 나 이런 사람들에게 AI를 가르치고 왔어요 라는 글들을 많이 보지만 반대로 궁금합니다.
조금 빨리 경험해본 것 외에 얼마나 튼튼하게 그 성을 지어가고 있나요? 빨리 무너질 모래성을 조금 더 빨리 쌓아보고 있는 것은 아닐까요?
