바이브코딩계 레전드 SU MIN CHOI님과 함께,
클로드 코드와 서브 에이전트를 활용하는 개발 방식을 촬영해보았어요.

무려 에이전트 100개를 다룬다고 하시네요. 모두 클로드코드 서브 에이전트를 사용하는 방식이었습니다. 주요 내용을 요약해보았습니다.

1. 서브에이전트로 컨텍스트 한계 돌파하기
클로드 코드의 핵심 강점은 '서브에이전트' 기능이에요. 커서나 다른 툴은 사용자가 직접 에이전트를 호출해야 하지만, 클로드 코드는 에이전트가 다른 에이전트를 호출할 수 있죠. 한 번의 명령으로 100개 이상의 작업을 병렬 처리 가능하고. 만드는 법도 간단합니다. claude/agents 폴더에 마크다운 파일을 만들고 name, description, tools, model만 정의하면 끝이더라고요.

2. 반복 작업은 스크립트로, LLM은 판단만
모든 걸 LLM에 맡기면 컨텍스트가 급격히 소모되어요. 예를 들어 스크린샷 생성을 플레이라이트 MCP로 하면 이미지 데이터가 컨텍스트에 그대로 쌓이죠. 해결책은 빈 깡통 파일 생성, 이미지 처리, HTML 파싱 같은 단순 반복 작업을 타입스크립트로 스크립트화하는 것이에요. 에이전트는 이 스크립트를 실행만 하고, 결과물만 받아서 다음 판단에 활용해요. 속도도 빠르고 정확도도 높지요.

3. 프롬프트를 아카이빙하면 에이전트가 된다
서브에이전트를 처음부터 설계하려 하면 어려워요. 대신 평소 쓰는 프롬프트를 파일로 모아두기를 추천해요. 반복해서 쓰게 되는 프롬프트가 보이면, 변수만 바꿔서 범용 에이전트로 전환해요. 예를 들어 "체크리스트 만들어줘" 프롬프트에서 특정 이름만 {{name}}으로 바꾸면 어떤 체크리스트든 만들 수 있는 에이전트가 되어요. 처음엔 불완전해도 괜찮아요. 결과를 보고 "폰트 웨이트 실수가 많네" 싶으면 해당 지침을 추가하며 점진적으로 개선하는 것이죠.

4. 파이프라인으로 대규모 작업 자동화하기
웹페이지 하나를 컴포넌트로 분해하는 작업을 예로 들면: URL만 입력하면 가상 브라우저가 스크린샷과 리소스를 추출하고, 섹션별로 분석한 뒤, 서브에이전트들이 병렬로 각 컴포넌트를 생성해요. 이 전체 흐름이 하나의 마크다운 파일에 지침으로 정의되어 있어요. 한 시간 정도 돌리면 125개 컴포넌트가 한 번에 구현됩니다. 커서에서는 컨텍스트 부족으로 불가능한 작업이죠.

5. 범용 도구는 MCP로 분리하라
자주 쓰는 스크립트 중 특히 범용적인 것은 MCP로 만들어 배포해요. 오픈소스 코드 조회, 나노바나 호출, 이미지 배경 제거 같은 기능이 그 예에요. MCP로 만들면 어떤 프로젝트에서든 에이전트가 바로 호출할 수 있고, 오픈소스로 공개하면 버그 제보나 수정도 받을 수 있어요. 핵심은 "중간 과정의 데이터가 컨텍스트에 남지 않게" 하는 것에요.
