#노코드 의 시대가 가고, 
해시태그
#바이브코딩 의 시대가 온다. 아니 왔다…

요즘 하루에 최소 하나 이상의 프로젝트를 만들어 배포하고 있습니다.

대부분 거창한 게 아닙니다. 데이터를 넣고 “대시보드 만들어줘”하면 끝. 순식간에 배포 완료. 이런 대시보드만 모아두는 GitHub 레포를 따로 관리하고 있을 정도입니다.

이것만이 아닙니다.

→ 주식 투자·재무 현황을 한눈에 보는 대시보드
→ 못 받은 전세금을 자동 계산해서 SMS까지 보내주는 도구
→ 회사 공동 청소 담당자를 랜덤 배정하고 Slack으로 미리 알림 보내는 시스템

전부 직접 딸깍해서 만들었고, 전부 실제로 쓰고 있습니다.

돌이켜보면 예전에 노코드를 썼던 이유는 하나였습니다. 개발이 너무 어려웠으니까.

그런데 지금은 노코드를 배우는 것보다 코딩이 더 쉽습니다.

누구나 만들 수 있는 시대. 차별점은 ‘만들 수 있느냐’가 아니라 ‘무엇을 왜 만드느냐’에서 나옵니다.

이런 흐름 속에서 기존 노코드 도구들, 나아가 다양한 SaaS 서비스들도 변화가 필요합니다. 더 작은 단위의 모듈로 쪼개지고, CLI나 MCP 형태로 제공되어야 살아남을 수 있습니다. 이제는 사람이 아니라 AI가 고객인 시대이니까요.

그래서 요즘은 오히려 만드는 시간보다, 무엇을 만들지 고민하는 시간이 더 길어졌습니다. 세상이 빨라질수록, 느리게 생각하는 힘이 더 중요해지는 걸 느끼고 있습니다.
