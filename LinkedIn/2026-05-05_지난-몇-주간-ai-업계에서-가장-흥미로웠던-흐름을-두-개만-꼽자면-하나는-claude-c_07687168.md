지난 몇 주간 AI 업계에서 가장 흥미로웠던 흐름을 두 개만 꼽자면, 하나는 Claude Code leak였고 다른 하나는 open agent stack 흐름이었다.

좋은 wrapper와 orchestration이 점점 보편화된다면, 기업의 다음 경쟁력은 어디서 나올까.
나는 그 답이 DSM(domain-specific model) 같은 도메인 특화 intelligence 쪽으로 이동할 가능성이 크다고 본다. 아니 이미 이렇게 옮겨 가고 있다.

둘은 겉으로 보면 다른 사건처럼 보인다.
하나는 유출이고, 하나는 공개다.

하지만 둘이 함께 보여준 건 비슷했다.

좋은 agent 경험을 만드는 구조가 점점 더 빨리 공개되고, 복제되고, 보편화될 수 있다는 점이다.

tool use, workflow orchestration, memory, context handling 같은 것들이 더 이상 소수 팀만의 비밀 병기가 아니라 점점 빠르게 퍼지는 영역이 되고 있다.

이 흐름이 계속되면 기업의 다음 질문도 바뀐다.

예전엔

• 어떤 foundation model을 쓸까?
가 중요했다면,

앞으로는
우리 회사만의 intelligence를 어디에 쌓을까?
가 더 중요해질 수 있다.

같은 foundation model을 써도 어떤 회사는 assistant를 얻고, 어떤 회사는 superpower를 얻는다.

그 차이는 점점 모델 access보다 도메인에 맞게 shape된 intelligence에서 나올 가능성이 크다.

이 지점에서 DSM(domain-specific model)이 다시 뜨거워지는 건 자연스럽다.

내가 여기서 말하는 DSM은 단순히 작은 fine-tuned model만 뜻하지 않는다.
특정 산업과 업무의 용어, workflow, decision rule, exception handling, risk tolerance, customer context를 학습한 특수한 intelligence layer에 더 가깝다.

Claude Code leak와 open agent stack 흐름이 보여준 건, 좋은 wrapper와 orchestration이 점점 commoditize될 수 있다는 점이다.

그렇다면 그 다음 차별화 포인트는 wrapper 자체보다 그 위에 어떤 intelligence를 쌓느냐로 이동한다.

RAG는 시작점일 수 있다.
하지만 경쟁이 깊어질수록 기업은 결국
“우리 도메인에서는 이렇게 판단해야 한다”
는 기준을 더 깊게 시스템 안에 심고 싶어질 것이다.

물론 DSM이 모든 기업의 정답은 아니다.
많은 기업은 여전히 범용 모델 + context layer + agent workflow만으로도 큰 효과를 볼 수 있다.

다만 지난 한 달의 흐름은 분명히 이런 질문을 더 중요하게 만들었다.

좋은 agent 구조가 보편화된다면,
우리 회사의 진짜 moat는 어디에 남는가?

내 생각엔 그 답이 점점 model access보다
domain-shaped intelligence, 즉 DSM 쪽으로 이동할 가능성이 크다.
