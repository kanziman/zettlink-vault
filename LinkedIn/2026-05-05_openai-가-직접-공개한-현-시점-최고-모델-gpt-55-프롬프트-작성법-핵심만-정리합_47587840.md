OpenAI 가 직접 공개한 현 시점 최고 모델 GPT-5.5 프롬프트 작성법.
핵심만 정리합니다.

프롬프트 가이드: https://lnkd.in/giDYAKdN

GPT-5.5 나오고 일주일 만에 공식 프롬프트 작성 가이드가 나왔습니다. 
한 줄로 요약하면 이겁니다. 
"과정을 지시하지 말고 결과를 정의하라." 

기존 프롬프트를 그대로 쓰면 오히려 성능이 떨어집니다. 
핵심 8가지로 정리했습니다.

❶ 기존 프롬프트를 버리고 새로 시작하세요 
이전 모델용 프롬프트를 그대로 가져오면 안 됩니다. 과정을 단계별로 지정하면 모델의 탐색 범위가 좁아지고 기계적인 답이 나옵니다. 
OpenAI 가 직접 경고하는 내용이에요. 최소한의 프롬프트에서 새로 시작하고 거기서 튜닝하라는 게 공식 권장 사항입니다.

❷ 결과를 정의하고 경로는 모델에게 맡기세요 
"먼저 A를 분석하고 B를 비교하고 C를 확인한 뒤 판단하라." 
이런 식의 단계별 지시가 GPT-5.5 에서는 노이즈입니다. 
대신 성공 기준을 명시합니다. 
"고객 문제를 처음부터 끝까지 해결하라. 완료 조건은 이 3가지다." 
이렇게 쓰면 모델이 스스로 경로를 찾습니다.

❸ 성격 설정과 협업 방식을 분리하세요 
이번 가이드에서 가장 눈에 띄는 구조입니다. 
성격 설정은 톤과 말투를 제어합니다. 따뜻하게 쓸지 간결하게 쓸지. 
협업 방식은 모델이 언제 질문하고 언제 가정하고 언제 멈추는지를 제어합니다. 둘을 섞어 쓰면 일관성이 떨어진다고 합니다.

❹ 사전 응답을 유도하면 체감 속도가 올라갑니다 
GPT-5.5 는 도구 호출이 많은 작업에서 첫 응답까지 시간이 걸립니다. 
"요청 확인했습니다. 먼저 ~부터 시작합니다" 같은 짧은 사전 응답을 프롬프트로 유도하면 사용자가 느끼는 대기 시간이 줄어듭니다. 
OpenAI 의 Codex 에서 이미 이 패턴을 쓰고 있어요.

❺ 멈추는 조건을 반드시 명시하세요 
GPT-5.5 는 자율성이 높아진 만큼 언제 멈춰야 하는지 모를 수 있습니다. 
"결과가 나올 때마다 스스로 물어라. 지금 핵심 질문에 답할 수 있는가? 가능하다면 답하라." 이런 중단 조건이 없으면 불필요한 도구 호출이 반복됩니다.

❻ text.verbosity 파라미터로 응답 길이를 제어하세요 
API 기본값은 medium 입니다. 짧은 응답이 필요하면 low 로 바꾸면 됩니다. 
프롬프트에 "간결하게 써라"를 반복할 필요가 없어졌어요. 
파라미터 하나로 제어하는 게 더 안정적입니다. 서식도 마찬가지입니다. 
기본 서식은 문단형이고 글머리표나 제목은 꼭 필요할 때만 쓰라고 안내합니다.

❼ 검색 횟수에 한도를 설정하세요 
GPT-5.5 에 웹 검색을 붙이면 검색을 과하게 돌리는 경우가 생깁니다. 
"일반 질의는 검색 1회로 시작하라. 추가 검색은 핵심 사실이 빠졌을 때만 하라." 이런 검색 한도를 프롬프트에 넣으면 토큰 소모를 크게 줄일 수 있습니다. 
표현을 다듬거나 부가적인 근거를 붙이려고 재검색하지 말라는 게 가이드의 원칙입니다.

❽ OpenAI 가 권장하는 프롬프트 구조가 있습니다 
역할 → 성격 → 목표 → 성공 기준 → 제약 조건 → 출력 형식 → 중단 규칙 
각 섹션은 짧게 씁니다. 
행동을 바꿀 수 있는 내용만 넣으세요. 
길어진다고 좋은 게 아닙니다.

⚠️ 한 가지 주의할 점 "반드시", "절대" 같은 절대 규칙은 안전 정책이나 필수 출력 필드에만 쓰세요. 판단이 필요한 영역에 절대 규칙을 쓰면 모델이 유연하게 판단할 기회를 빼앗습니다. OpenAI 는 이걸 "의사 결정 규칙으로 대체하라"고 권합니다.

GPT-5.5 가 빨라지고 똑똑해진 건 맞는데 프롬프트를 바꾸지 않으면 체감이 안 됩니다. 기존 프롬프트를 쓰고 있다면 이 가이드 기준으로 한번 다시 써보세요.
