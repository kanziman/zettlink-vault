OpenClaw에서 Hermes Agent로.... (몇 가지 활용법과 함께)
.
.
.
이건 Agent 하나를 바꾼게 아니라 "일하는 방식이 바뀐" 느낌이다.

Hermes는 Nous Research(Hermes, Nomos 모델로 유명한 그 연구소)가 만든 오픈소스 로컬 에이전트다. 

**Hermes는 Nous Research가 만든 오픈소스 로컬 에이전트다. Nous Research는 오픈소스 AI 모델 Hermes 시리즈로 유명한 미국 AI 연구소로, Paradigm 주도로 $70M을 투자받은 곳이다. 오픈소스 LLM 커뮤니티에서는 모르는 사람이 없을 정도.

한마디로 Hermes 는 나와 함께 성장하는 AI랄까? 

OpenClaw보다 좋다고 느끼는 이유는 다음과 같다:

- 반복 작업을 알아서 Skill로 만들어 준다. 
- Skill이 완벽해질 때까지 나에게 물어보며 더 좋은 시스템을 완성하려 한다.
- Telegram 연동이 훌륭하다 — 위험 명령은 승인 요청, 메시지도 버튼형식으로 자연스럽게 끊어서 보여준다
- 진행 중인 작업을 실시간 스트리밍으로 보여줘서 가시성이 좋다.
- 메모리 크기도 관리해서 Context Window를 효율적으로 운영한다.
- 웹 대시보드로 전체 설정과 세션을 시각적으로 관리할 수 있다.
- 멀티 에이전트를 만들어서 각 Agent에게 업무를 위임하고, 결과를 종합할 수 있다 (지금 한창 세팅 중인데 하네스 구성이 고민될 뿐 굉장히 흥미롭다.)
- 코딩은 Claude Code나 Codex를 별도로 불러서 맡길 수 있다.
- 뭐 느낌적으로 지금까지 OpenClaw보다 알잘딱깔센 한 느낌이다.

OpenClaw는 며칠간 열심히 세팅하다 어느순간 생각보다 안똑똑해서 흥미가 확떨어졌는데, 이 친구는 뭘 더 해볼 수 있을지 계속 고민하게 된다. 

그리고 하나 더 공유. OpenClaw든 Hermes든 MCP 연동이 제대로 안되거나 메모리 관리 등이 잘 안 될 때가 있다. 이때 채팅으로 열심히 이야기 해도 해결이 요원해지거나 너무 오랫동안 대화해야 하는 문제가 있다. 

이 문제를 더 빨리 해결하는 방법을 찾았다:

1. IDE에서 해당 Agent 폴더를 연다 (예: ~/.hermes)
2. Claude Code 플랜 모드로 문제를 분석한다 (관련 문서까지 읽도록 요청)
3. 핵심: Claude Code가 직접 고치지 않고, Agent에게 알려줄 수정 프롬프트를 짜달라고 한다

왜 이렇게 하냐면, Agent들이 아직 웹 검색이 약하다. Bot 검색이 차단되는경우도 많다. 그런데 Claude는 이걸 더 잘한다. 반대로 발견된 문제를 Claude Code가 직접 수정하면 OpenClaw/Hermes Agent가 수정된 내용에 대한 컨텍스트를 잃는다. 어디가 바뀐 건지 히스토리도 없고, 뭔가 잘못되면 되돌리기도 어렵다.

그래서 Agent 자신이 방법을 전달받아 직접 고치면? 자기 맥락 안에서 해결하니까 훨씬 안정적이다. 구지 비유하자면 마치 회사에서 다른 팀이 우리 시스템을 직접 수정해주는 것과, 우리 팀에게 해결 방법만 알려주는 차이랄까? 

현재 나는 Mac Mini 한 대를 24시간 켜두고, Hermes와 Claude Cowork/Code의 스케줄을 동시에 돌리고 있다. 참고로 Cowork 스케줄은 컴퓨터가 깨어있어야 실행된다. Mac Mini가 단순 장비가 아니라 자동화의 핵심 인프라가 되는 이유다.

Claude Code/Cowork 루틴:
- Notion, Slack, Mem.ai를 1시간마다 점검하고 나의 통합 대시보드 업데이트. 이 노트 하나만 보면 회사가 어떻게 돌아가는지, 뭘 점검하고 실행해야 하는지 한눈에 파악할 수 있다.
- 매일 최신 생산성 도구 정보를 당근메일 커뮤니티에 자동 웹훅 포스팅
- 하루 동안 작업한 채팅 내용과 노트를 모아서 Daily Record로 자동 정리

Hermes 루틴:
- 매일 아침 Daily Scrum 초안 자동 생성
- 매일 아침 내 주식 포트폴리오 기반 관련 뉴스 브리핑 및 매수/매도 인사이트 제공
- 하루 3번 이메일을 읽고 긴급 답변이 필요한 메일 확인 및 뉴스레터 정보 요약

하루하루 자동화 작업을 늘리면서 Mac Mini 200% 활용 중이다.
요즘 중요한 건 어떤 Agent를 쓰느냐가 아니라 그 Agent와 어떻게 함께 일하느냐인 것 같다. 구축하는건 어짜피 Agent 가 다 해주니 아이디어를 공유하는 공간이 필요해졌다. 지금은 이렇게 하나 둘 LinkedIn으로 공유하지만 여전히 배고프다... 

뭐 암튼, 나는 요즘 아침에 출근하면 이미 스크럼이 써져 있고, 이메일이 정리되어 있고, 뉴스 브리핑이 와 있다. 내가 자는 동안에도 Mac Mini 위의 AI들은 일하고 있다. 그리고 이제 나는 나의 '쓸모'를 찾기 위해 내 진짜 일을 만들어 내야 한다. (아.. 이게 더 어렵네 ㅋㅋ)
