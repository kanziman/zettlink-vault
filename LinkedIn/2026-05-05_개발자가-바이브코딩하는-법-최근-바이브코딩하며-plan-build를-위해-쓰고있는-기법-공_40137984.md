[개발자가 바이브코딩하는 법]

최근 바이브코딩하며 plan + build를 위해 쓰고있는 기법 공유드립니다.

1. LLM과 충분히 대화하며 양질의 context 축적
2. task-phases로 구성된 구현계획 작성
3. python script 활용해 직렬 claude session으로 실행

---

📌 특징
1. 먼저 작업에 대해 최대한 구체적으로 논의합니다.
2. 여러 phase로 나눕니다. 테스트를 고려해 userflow/큰모듈 단위로 나눕니다.
3. 구현 전, spec 문서를 먼저 업데이트합니다.
4. 업데이트 diff 를 참조하며 순차적으로 구현합니다.

📌 이 방식의 장점
반복작업(git, orchestrating)을 python script에 맡기고, 각 claude code 세션은 중요 코딩 작업만 수행합니다. 이로 인해 여러 부수효과가 생깁니다.

1. 속도가 빨라집니다.
2. LLM context에 core한 내용만 남기 때문에, 정확도가 올라갑니다.
3. 토큰 소모가 줄어듭니다. 비용도 낮아집니다.
4. orchestrating 신뢰도가 높아져, 많은 병렬 작업도 안정적으로 완수합니다.
5. spec 문서를 꼭 업데이트하기 때문에, 문서-코드 sync 문제가 꽤 해소됩니다.

---

개인적으로 LLM과 협업할때의 핵심은, LLM의 context 대신 cli/script등으로 programmatic하게 해결할 수 있는 것들을 최대한 덜어내는 것이라고 생각합니다. testcode, linter 등도 여기 해당하구요.

이걸 극한으로 깎으면 파이프라인 코딩이 됩니다!
